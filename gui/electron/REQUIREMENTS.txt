## WHY THESE CHOICES
# ─────────────────────────────────────────────────────────────────────────────

node-pty vs child_process.spawn:
  node-pty creates a real PTY (pseudo-terminal), which means PowerShell
  outputs ANSI escape codes, progress bars, and interactive prompts correctly.
  child_process.spawn would strip colours and buffer output until the process
  exits — you'd get no live streaming.

xterm vs a plain <pre> element:
  xterm handles ANSI/VT100 escape codes from PowerShell output correctly.
  It renders the EntraScope coloured log lines (cyan/green/yellow/red) exactly
  as they look in a real terminal.

electron-store vs localStorage:
  Persists config across app restarts. localStorage is cleared when the app
  is rebuilt; electron-store writes to %APPDATA%\EntraScope\config.json.

chart.js vs D3.js:
  Chart.js requires ~200 lines of config. D3.js would require ~800+ lines
  for the same charts. Chart.js is sufficient for the score/phase visuals.


## INSTALL COMMANDS
# ─────────────────────────────────────────────────────────────────────────────

# 1. Install Node.js LTS from https://nodejs.org then verify:
node --version    # must be >= 20
npm --version     # must be >= 10

# 2. Install Windows build tools (for node-pty native compilation)
npm install --global windows-build-tools
# OR open Visual Studio Installer → Modify → Desktop development with C++

# 3. Install dependencies
cd EntraScope\gui\electron
npm install

# 4. Start in dev mode
npm run dev

# 5. Build distributable installer
npm run build
# Output: EntraScope\gui\electron\dist\EntraScope-Setup-1.0.0.exe
#         EntraScope\gui\electron\dist\EntraScope-1.0.0-portable.exe

## TROUBLESHOOTING NODE-PTY BUILD ERRORS
# ─────────────────────────────────────────────────────────────────────────────

Error: "gyp ERR! find Python"
  → Install Python 3.x from python.org, add to PATH, then retry npm install

Error: "MSBuild not found"
  → Install Visual Studio Build Tools with C++ workload (see above)

Error: "node-pty failed to load"
  → Run: npm rebuild node-pty
  → Or: npm install --build-from-source node-pty

After Electron version upgrade:
  → Run: npm run rebuild   (rebuilds native modules against new Electron ABI)
